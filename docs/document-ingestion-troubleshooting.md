# Troubleshooting: อัปโหลด Word/PDF แล้วค้นหาไม่เจอ

## เช็คสถานะเอกสารที่อัปโหลด
- ไปที่หน้า `law-chatbot/upload` แล้วดูสถานะว่าเป็น `completed` หรือ `quarantined`
- ถ้าเป็น `quarantined` ระบบจะไม่เอาเอกสารนั้นไปใช้ค้นหา (มักเกิดจากอ่านข้อความไม่ได้ หรือคุณภาพข้อความต่ำมาก)

## กรณีอัปโหลดไฟล์ Word (DOC/DOCX) แล้วมี `chunk` น้อยผิดปกติ
- DOC/DOCX บางไฟล์เป็นข้อความสั้น/หัวข้อ/รายการแบบ bullet จำนวนมาก ทำให้คะแนน “คุณภาพข้อความ” ต่อ chunk ต่ำและถูกคัดทิ้งได้
- ตั้งค่าเกณฑ์สำหรับ DOC/DOCX ให้ผ่อนกว่า PDF ได้ด้วยตัวแปรใน `.env`
  - `DOCUMENT_CHUNK_ACCEPTED_QUALITY_SCORE` (ค่าแนะนำเริ่มต้น `45`)
  - `DOCUMENT_CHUNK_LOW_QUALITY_SCORE` (ค่าแนะนำเริ่มต้น `35`)

## หลังปรับโค้ด/ปรับค่า .env แล้วต้องทำอะไรต่อ
- เอกสารที่นำเข้าไปก่อนหน้า *จะไม่ถูก re-index อัตโนมัติ*
- วิธีที่ง่ายสุดคืออัปโหลดไฟล์เดิมใหม่อีกครั้ง เพื่อให้ระบบสร้าง chunks ใหม่ด้วย logic ใหม่

## หมายเหตุเรื่อง Embeddings
- ถ้า `ENABLE_EMBEDDING=false` การค้นหาจะพึ่ง keyword match เป็นหลัก
- ถ้าต้องการผลค้นหายืดหยุ่นขึ้น (semantic) ให้เปิด embeddings และรันสคริปต์สร้าง embeddings ตามที่โปรเจกต์เตรียมไว้
